Nyzr terminal

Organize any folder with rules.

Download this project as a .zip file Download this project as a tar.gz file

Welcome to NYZR Terminal.

This is a current work in progress.

Inspiration & Goal:

For every hacker to have a clean downloads folder.

Get Started

pip install -r requirements
python nyzr.py

How it works

Each folder should be monitored with a set of rules (see .nrules file), all of your rules should be kept in a json encoded file.

I designed nyzr with modularity in mind

"filters":{
        "pdf_filter":{
            "extension":"*.pdf"
        },
        "small_size_filter":{
            "size":"<1"
        },
        "large_size_filter":{
            "size":">200"
        },
        "movie_filter":{
            "regex":"(?=.*(?:\\.mpeg|\\.mov|\\.mp4|\\.flv|\\.avi|\\.w4v|\\.rmvb|\\.qt|\\.mwv|\\.mpg)$).*$"
        },

    },

Filters are separate simple rules that you can set by regex, extension, and size (more on the way :)).

Operations are the actions taken once your files meet the rules. 

"operations":{
        "pdf_operation":[
                    {
                        "copy":"~/PDFs"
                    },
                    {
                        "copy":"~/PDFs_backup"
                    },
                    {
                        "delete":""
                    }
                ],
        "movie_operation":[
                    {
                        "move":"~/my_movies"
                    }
        ]
    }

Lastly, we have directories, which is just a container for different rules and operations. While the order for rules do not matter, it does for operations. 

"directories":[
        {
        "directory_path":["~/Downloads/"],
        "rules":[
            {
                "filters":["pdf_filter","small_size_filter"],
                "operations":["pdf_operation"]
            },
            {
                "filters":["movie_filter","large_size_filter"],
                "operations":["movie_operation"]
            }
        ]
        }
    ]
